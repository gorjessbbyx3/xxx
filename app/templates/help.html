Overview

GhostTrace Dark Web Crawler is a powerful tool designed for exploring, crawling, and analyzing content from the dark web (.onion sites) with advanced AI-powered analysis capabilities.

Important: This tool should only be used for legitimate security research, threat intelligence gathering, and educational purposes. Always ensure compliance with applicable laws and regulations in your jurisdiction.

The main features of GhostTrace include:

  • Dark Web Crawling: Safely crawl .onion sites via the Tor network
  • AI Analysis: Analyze content using the Ollama LLM for entities, topics, and risk assessment
  • Security Tools: Integration with various security and penetration testing tools
  • Anonymity: All traffic routed through Tor for privacy and access to .onion sites

Requirements

For GhostTrace to function properly, you need the following services running:

Tor Service

The Tor service is required to access .onion sites. To start Tor:

sudo service tor start

Ollama Service

Ollama provides the AI analysis capabilities. To start Ollama:

ollama serve

Then pull the necessary model (if not already available):

ollama pull mistral:7b-instruct

Dark Web Crawling

To start a new crawl:

  1. Navigate to the main dashboard
  2. Enter one or more .onion URLs in the crawl form
  3. Set the crawl depth (how many links deep to follow)
  4. Set the maximum number of pages to crawl
  5. Enable or disable AI analysis
  6. Click "Start Crawling"

Crawl Parameters

  • Depth: How many links deep to follow from the starting URLs. A depth of 0 means only the initial URLs will be processed.
  • Max Pages: Maximum number of pages to crawl in total
  • Delay Range: Time range (in seconds) to wait between requests to avoid detection

Viewing Results

Results are stored in the crawl_results directory, organized by job ID and domain. Each crawled page has:

  • The original HTML content
  • Extracted text content
  • AI analysis results (if enabled)

AI Analysis

When AI analysis is enabled, GhostTrace uses the Ollama LLM to perform the following analyses:

  • Content Categorization: Identifies the primary topics and themes in the content
  • Entity Extraction: Identifies people, organizations, locations, cryptocurrencies, emails, etc.
  • Risk Assessment: Evaluates the content for potential risks and illegal activities

Single URL Analysis

To analyze a single URL without crawling its links:

  1. Go to the "Analyze Single URL" section
  2. Enter a valid .onion URL
  3. Click "Analyze URL"

This will fetch the URL content and perform AI analysis without following any links.

Security Tools

GhostTrace integrates with a wide range of security tools for comprehensive dark web research and security testing.

Tool Categories

  • Information Gathering: Tools for reconnaissance and information collection
  • Wireless Attack Tools: Tools for testing and auditing wireless networks
  • Web Attack Tools: Tools for testing web application security
  • Phishing Tools: Social engineering and phishing simulation tools
  • Exploitation Tools: Frameworks for testing vulnerabilities
  • Post Exploitation: Tools for post-compromise assessment
  • Forensic Tools: Digital forensics and analysis tools

Visit the Security Tools section to access these capabilities.

API Reference

GhostTrace provides a REST API for programmatic access:

Endpoints

  • GET /api/status - Check service status (Tor, Ollama)
  • GET /api/jobs - List all crawler jobs
  • GET /api/jobs/{job_id} - Get status of a specific job
  • POST /api/crawl - Start a new crawling job
  • POST /api/analyze - Analyze a single URL

Example: Starting a Crawl via API


curl -X POST http://localhost:5000/api/crawl \
  -H "Content-Type: application/json" \
  -d '{
    "urls": ["http://example.onion"],
    "depth": 1,
    "max_pages": 10,
    "analyze": true
  }'
